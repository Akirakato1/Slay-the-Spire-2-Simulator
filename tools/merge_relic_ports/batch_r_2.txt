// Relic-effect data table — batch_r_2
//
// Second-pass encoding, run after the Effect VM + RelicHook vocabulary was
// expanded with:
//   * AfterCombatEnd, AfterCardPlayed, BeforeTurnEnd, AfterDamageGiven,
//     AfterDamageReceived hooks
//   * AddRandomCardFromPool / AutoplayCardsFromPile / SummonOsty / HealOsty
//   * AmountSpec::TargetMaxHp / TargetBlock / TargetDebuffCount /
//     CardsPlayedThisTurn / CardsDiscardedThisTurn / EnergySpentThisTurn /
//     OrbsChanneledThisCombat
//   * Condition::IsOstyMissing / RoundEquals{n} / RoundGe{n} /
//     OwnerExhaustedCardThisTurn / PlaysThisTurnLt{n}
//
// Re-encodes the previously-SKIPped relics in batch_r_1.txt where the new
// vocabulary now suffices for a runtime-faithful body. Same emit conventions
// as batch_r_1 (canonical-var keys, owner_side_only, etc.).
//
// Counter-state caveat: many relics gate their effect body on an internal
// "every Nth event" counter (Kunai/Shuriken/Nunchaku/IronClub/LetterOpener
// etc.). The current Effect VM has no IncrementRelicCounter primitive, so
// these arms fire on EVERY matching event rather than every Nth. Each such
// arm is tagged "counter-ungated" in the evidence comment.
