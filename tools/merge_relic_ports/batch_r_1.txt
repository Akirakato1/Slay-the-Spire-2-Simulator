// Relic-effect data table — batch_r_1
//
// Survey of 294 relic .cs files in
//   sts2_decompiled/.../Core/Models/Relics/
// against the RelicHook enum:
//   BeforeCombatStart | AfterCombatVictory | AfterCombatLoss
//   AfterSideTurnStart { owner_side_only, first_turn_only }
//   BeforeSideTurnStart { owner_side_only, first_turn_only }
//   AfterPlayerTurnStart { first_turn_only }
//   AfterPlayerTurnEnd
//
// Encoded = single-Cmd body whose primitive already exists in the Effect VM
// vocabulary (effects.rs ~lines 47-535) AND uses only one of the above hooks.
//
// Conventions used here:
// * `owner_side_only: true` when guard is `side == base.Owner.Creature.Side`
//   (and `false` when guard is `side == CombatSide.Player` literal, which the
//   C# uses interchangeably for player-owned relics — same semantics here).
// * `first_turn_only: true` when guard is `combatState.RoundNumber <= 1` or
//   `== 1` (the most common idiom; <=1 vs ==1 difference is cosmetic
//   pre-turn-1).
// * `RoundNumber > 1`, `RoundNumber == 2`, `RoundNumber == N` for fixed N>1,
//   or `RoundNumber >= N` for N>1 are NOT covered by `first_turn_only`
//   — those go to SKIP with a "needs Round=N gate" reason.
// * GainBlock on `Owner.Creature` uses `BlockVar(...,Unpowered)` in many
//   relics. We encode it as a plain `Effect::GainBlock` (Unpowered prop
//   lost; matches cookbook guidance).
// * Already-in-dispatcher relics (Anchor, BurningBlood, Brimstone) are NOT
//   re-emitted here; they live as match-arm code in combat.rs.
